# 项目面试讲解提纲

## 一句话介绍

CareerPilot 是一个基于 Spring Boot + Vue3 的 AI 求职助手平台，支持简历解析、岗位匹配分析、模拟面试、AI 对话、模板化简历编辑和历史记录管理。

## 技术架构

```text
Vue3 + Element Plus
        ↓ Axios
Spring Boot RESTful API
        ↓
Service 业务层
        ↓
JPA / H2 / Redis / Apache Tika / OpenAI 兼容接口
```

## 后端分层

- `Controller`：接收 HTTP 请求，处理参数和登录上下文。
- `Service`：实现简历解析、AI 调用、历史记录、验证码、认证等业务逻辑。
- `Repository`：通过 Spring Data JPA 访问数据库。
- `DTO`：封装请求和响应数据。
- `GlobalExceptionHandler`：统一处理异常并返回标准结构。

## 核心业务链路

### 1. 简历解析

1. 前端上传 PDF/DOCX/TXT 文件。
2. 后端检查文件类型和大小。
3. 使用 Apache Tika 提取文本。
4. 返回纯文本给前端。
5. 文本作为 AI 分析和简历编辑器导入的基础。

### 2. AI 分析

1. 前端提交简历文本、岗位 JD 和可选 AI 配置。
2. 后端组装 Prompt。
3. WebClient 调用 OpenAI 兼容接口。
4. 将模型返回内容解析成结构化结果。
5. 保存 `analysis_record`。
6. 返回匹配分数、优势、短板和建议。

### 3. 历史记录

1. 登录用户请求会携带 JWT。
2. 后端解析 Token 得到用户 ID。
3. 保存记录时写入 `user_id`。
4. 查询历史时按 `user_id` 过滤。
5. 支持详情恢复和删除。

### 4. 验证码登录

1. 用户输入手机号并发送验证码。
2. 后端生成 4 位字母数字验证码。
3. 优先写入 Redis，设置过期时间。
4. Redis 不可用时写入本地内存缓存。
5. 注册或重置密码时校验验证码。

## 可以重点强调的技术点

- 使用 Apache Tika 统一解析多格式简历文件。
- 使用 Spring Validation 和文件校验保证输入安全。
- 使用 WebClient 封装 AI 调用，增加超时和异常兜底。
- 使用统一返回结构和全局异常处理提升接口规范性。
- 使用 JPA 保存分析记录，并按用户维度隔离数据。
- 使用 Redis 缓存验证码，并提供本地缓存降级。
- 使用 JWT 实现轻量登录态。
- 前端支持用户自定义 AI 中转站配置，提高项目实用性。
- 简历编辑器支持模板、照片、主题、字体、多格式导出，产品完整度较高。

## 可能被问到的问题

### 1. 你的项目难点是什么？

可以回答：

> 难点主要有三个。第一是 AI 返回内容不稳定，需要设计结构化 Prompt 和 JSON 解析兜底；第二是文件解析要兼容 PDF、DOCX、TXT，所以使用 Apache Tika 统一处理；第三是演示稳定性，AI 接口失败、Redis 不可用、用户未配置 Key 等情况都要有降级方案。

### 2. 为什么历史记录要绑定用户？

可以回答：

> 因为简历属于个人敏感数据，登录后用户只能查看自己的分析记录。实现上是 JWT 解析出用户 ID，保存分析记录时写入 `user_id`，查询和删除时都按用户 ID 做权限过滤。

### 3. 怎么避免 API Key 泄露？

可以回答：

> 项目默认 Key 放在后端 `.env`，不会提交到 GitHub。用户自己的 Key 在前端配置，只保存在浏览器本地，并且由后端作为代理发起请求。后续上线还可以对用户 Key 做加密存储或只支持临时会话级使用。

### 4. 如果要上线，你会怎么改？

可以回答：

> 我会把 H2 切换为 MySQL，增加 Docker Compose，配置 Nginx 反向代理；验证码接入真实短信服务；AI 调用增加用户额度限制；日志接入文件或监控系统；同时把 `.env`、数据库文件和日志文件排除在 GitHub 外。

### 5. 项目和普通 CRUD 有什么区别？

可以回答：

> 它不是单纯 CRUD，而是包含文件解析、AI 调用、用户认证、历史记录、前端复杂交互和多格式导出。核心价值在于把简历分析、面试准备和简历制作串成完整业务闭环。

## Java/Spring Boot 复习重点

- Java 集合：`ArrayList`、`HashMap`、扩容、哈希冲突。
- 异常处理：受检异常、运行时异常、全局异常处理。
- IO：文件上传、输入流、文本解析。
- 多线程基础：线程、线程池、异步任务概念。
- Spring Boot：Controller/Service/Repository 分层。
- 依赖注入：Bean、`@Service`、`@Repository`、构造器注入。
- JPA：实体、主键、自增、Repository 查询。
- Redis：缓存验证码、过期时间、降级方案。
- JWT：Token 生成、携带、解析、用户身份识别。
