# 招聘会演示脚本

## 30 秒项目介绍

CareerPilot 是我做的一个 AI 简历分析与模拟面试平台，主要面向大学生求职场景。用户可以上传简历并输入目标岗位 JD，系统会自动解析简历内容，调用大模型生成岗位匹配分数、技能短板、优化建议和模拟面试题。同时我还做了登录注册、历史记录、AI 对话和模板化简历编辑器，形成了从“做简历”到“改简历”再到“准备面试”的完整流程。

## 3 分钟演示流程

### 1. 登录注册

话术：

> 这里是登录注册入口。我实现了手机号验证码注册、忘记密码和 JWT 登录。验证码优先走 Redis 缓存，Redis 不可用时会自动降级到本地缓存，方便本地演示。

演示动作：

- 点击“登录 / 注册”。
- 切换注册。
- 输入手机号并发送验证码。
- 展示验证码倒计时。

### 2. 简历与 JD 输入

话术：

> 左侧是简历和岗位输入区。简历支持 PDF、DOCX、TXT 上传，后端用 Apache Tika 做文本解析；也可以直接粘贴简历文本。JD 会作为 AI 分析和模拟面试的上下文。

演示动作：

- 上传示例简历或粘贴简历。
- 粘贴 Java 后端实习 JD。

### 3. AI 简历分析

话术：

> 点击开始分析后，后端会把简历和岗位 JD 组装成 Prompt，通过 OpenAI 兼容接口调用大模型。返回结果会被解析成结构化数据，包括匹配分数、优势、短板、建议和项目改写。AI 异常时会返回本地兜底结果，避免演示中页面不可用。

演示动作：

- 点击“开始分析”。
- 展示匹配分数、优势、短板和优化建议。

### 4. 模拟面试

话术：

> 右侧可以根据当前简历和 JD 生成模拟面试题，帮助我准备项目讲解和技术追问，比如 Java、Spring Boot、MySQL、Redis 和 AI API 调用。

演示动作：

- 点击“生成面试题”。
- 展示题目列表。

### 5. 历史记录

话术：

> 每次分析都会保存历史记录。登录用户只能看到自己的记录，支持详情查看、恢复现场和删除，这里体现了用户维度的数据隔离。

演示动作：

- 点击右上角“最近记录”。
- 打开详情并恢复。
- 删除一条测试记录。

### 6. 简历编辑器

话术：

> 这个模块是模板化简历编辑器，类似一个轻量版的在线简历制作工具。用户可以切换模板、直接在 A4 简历上编辑文字、上传照片、修改主题色和字体，还能导出 PDF、HTML、JSON、Markdown 和 WPS 可编辑 Word。

演示动作：

- 打开简历编辑器。
- 切换模板。
- 修改姓名或项目经历。
- 上传照片。
- 调整缩放。
- 导出 Word 或 PDF。

### 7. AI 配置

话术：

> AI 配置支持两种方式。后端 `.env` 可以配置默认模型；用户也可以在前端填写自己的中转站 Key、Base URL 和模型名。用户配置后，简历分析、面试题和 AI 对话会优先使用用户自己的配置。

演示动作：

- 点击右上角 AI 状态按钮。
- 展示配置项。

## 常见追问回答

### Q1：为什么用 H2，不直接用 MySQL？

为了本地演示和提交方便，我默认使用 H2 文件数据库，启动成本低。但实体模型和 JPA 持久化是按关系型数据库设计的，后续切换 MySQL 只需要引入 MySQL 驱动、修改数据源配置，并补充索引。我已经准备了 `app_user` 和 `analysis_record` 的 MySQL 建表方案。

### Q2：AI 调用失败怎么办？

后端对 AI 调用做了超时控制和异常捕获。如果模型接口不可用，会返回本地兜底结果，保证页面不会崩掉，也方便招聘会现场演示。

### Q3：API Key 会不会泄露？

后端默认 Key 放在 `.env`，不会提交到 GitHub。用户前端填写的 Key 保存在浏览器本地，并通过后端转发调用，不写死在前端代码里。

### Q4：登录状态怎么做的？

注册登录成功后，后端生成 JWT Token。前端请求用户相关接口时在 `Authorization` 里携带 Token，后端解析 Token 获得用户 ID，再进行历史记录隔离。

### Q5：验证码为什么直接显示在页面上？

这是演示模式，方便本地测试。真实上线时可以接入短信服务，把验证码发到用户手机，不在页面展示。
